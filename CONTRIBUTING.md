# Contributing

## Setup

```bash
git clone https://github.com/cfpmonitor/CFPMonitor.git
cd CFPMonitor

conda env create -f environment.yaml
conda activate cfpmonitor

pip install -e ".[dev]"
```

## Architecture

The pipeline has three stages, each a separate module under `src/`:

```
cfpmonitor.py  (CLI orchestrator)
    │
    ├── src/crawler/     crawl  →  data/raw/, data/ccfddl/, data/wikicfp/
    ├── src/extractor/   extract → data/structured/
    └── src/site/        build   → docs/
```

### Crawler (`src/crawler/`)

| File | Role |
|---|---|
| `base.py` | `BaseCrawler` -- shared HTTP fetching, rate limiting, caching |
| `conf.py` | `ConfCrawler` -- BFS crawl of conference websites, saves HTML to `data/raw/` |
| `ccfddl.py` | `CCFDDLCrawler` -- fetches structured data from ccf-deadlines GitHub |
| `wikicfp.py` | `WikiCFPCrawler` -- scrapes WikiCFP event pages |
| `cache.py` | File-based HTTP cache (`data/.cache/`) |
| `core.py` | Standalone `fetch_html()` for one-off requests |

All crawlers inherit from `BaseCrawler`, which handles encoding detection, content-type filtering, and rate limiting.

### Extractor (`src/extractor/`)

Three-phase pipeline in `extractor.py`:

1. **Collect** -- loads `data/raw/` text files + wikicfp `cfp_text` into a unified text corpus
2. **Extract** -- runs regex patterns (`patterns.yaml`) against the corpus, guided by field definitions (`fields.yaml`)
3. **Merge** -- applies structured overrides from ccfddl/wikicfp with priority (ccfddl > wikicfp > regex), detects conflicts, calculates quality metrics

`helpers.py` contains type-driven value extraction (date parsing, enum matching, boolean detection, etc.).

**To add a new extraction field:**

1. Add the field definition to `fields.yaml` (type, category, priority, default)
2. Add regex patterns to `patterns.yaml`
3. If the field has a structured source (ccfddl/wikicfp), add the mapping to `_STRUCTURED_FIELD_MAP` in `extractor.py`

### Site Builder (`src/site/`)

`build.py` reads `data/structured/` YAML files and `data/metadata/conferences.yaml`, then renders Jinja2 templates into `docs/`. The frontend is vanilla JS with a masonry card layout.

| File | Role |
|---|---|
| `build.py` | Main build logic -- loads data, renders templates, writes `docs/index.html` |
| `renderer.py` | Jinja2 template rendering |
| `generate_schema.py` | Generates `docs/js/generated_schema.js` from field definitions |
| `templates/` | Jinja2 templates for page sections |

**Frontend modules** (`docs/js/`):

| File | Role |
|---|---|
| `app.js` | Entry point -- initializes filters, loads data, renders cards |
| `cards.js` | Conference card rendering |
| `rules.js` | Rule tag rendering with popovers |
| `filters.js` | Search, area, rank, and date filtering |
| `state.js` | Global state management |
| `masonry-manager.js` | Masonry layout initialization and updates |
| `constants.js` | CSS class names, error messages |
| `utils.js` | Date formatting, HTML escaping |

## Data Layout

```
data/
├── metadata/
│   ├── conferences.yaml          # 321 conferences (short, name, area, rank)
│   ├── conferences_homepage.yaml # homepage URLs (auto-updated by ccfddl crawler)
│   └── areas.yaml                # 10 area definitions
├── raw/{conf}/{year}/            # archived HTML (gitignored)
├── ccfddl/{conf}/{year}.yaml     # ccf-deadlines data (gitignored)
├── wikicfp/{conf}/{year}.yaml    # WikiCFP data (gitignored)
└── structured/{conf}/{year}.yaml # extracted rules (gitignored)
```

Only `data/metadata/` is committed. All other data directories are generated by the pipeline and gitignored.

## Common Tasks

### Add a new conference

Add an entry to `data/metadata/conferences.yaml`:

```yaml
- short: CONF
  name: Full Conference Name
  area: AI          # one of: AI, SE, SC, DB, NW, DS, CG, HI, CT, MX
  rank:
    ccf: A          # optional
    core: A*        # optional
    thcpl: A        # optional
```

Then run: `python cfpmonitor.py all --conference CONF --year 2026`

### Modify extraction patterns

Edit `src/extractor/patterns.yaml`. Each field has a list of regex patterns tried in order (first match wins). Test with:

```bash
python cfpmonitor.py extract --conference ICML --year 2026
# check data/structured/icml/2026.yaml
```

### Build the site locally

```bash
python cfpmonitor.py build
python -m http.server -d docs 8000
# Visit http://localhost:8000
```

### Clear cached data

Delete `data/.cache/` to force re-fetching from remote sources. Delete `data/wikicfp/` or `data/ccfddl/` to re-crawl those sources.

## Code Style

- **Formatter/Linter**: `ruff` (handles both linting and formatting)
- **Docstrings**: reStructuredText (`:param:`, `:return:`)
- **Type hints**: required for public functions

```bash
ruff check .
ruff format .
pytest
```

## Commit Messages

Use imperative mood. Prefix with `feat:`, `fix:`, `refactor:`, `docs:`, or `chore:`.

```
feat: add LLM policy extraction for ICML
fix: handle missing charset in WikiCFP responses
```
